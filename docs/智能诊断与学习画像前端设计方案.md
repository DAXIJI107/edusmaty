# 智能诊断与学习画像前端设计方案

## 1. 需求分析

### 1.1 现状问题

- 后端已实现完整的智能诊断系统（BKT知识追踪、CAT自适应测试、DINA认知诊断、误区检测）
- 前端缺少与之匹配的可视化界面和交互功能
- 现有诊断页面功能单一，仅支持基础问卷和学科测试
- 学习画像展示信息有限，缺乏深度数据可视化

### 1.2 用户期望

- **学生视角**：直观了解自己的知识掌握情况、认知能力、学习建议
- **交互体验**：流畅的自适应测试流程、实时反馈、清晰的进度展示
- **数据可视化**：雷达图、热度图、金字塔等专业图表展示

### 1.3 后端API支撑（已实现）

| API端点                               | 方法 | 功能描述               |
| ------------------------------------- | ---- | ---------------------- |
| `/api/diagnostic/vark-questionnaire`  | GET  | 获取VARK学习风格问卷   |
| `/api/diagnostic/smart-start`         | POST | 启动CAT自适应测试      |
| `/api/diagnostic/smart-next-question` | POST | 获取下一道自适应题     |
| `/api/diagnostic/smart-submit-answer` | POST | 提交答案并更新能力估计 |
| `/api/diagnostic/smart-status`        | GET  | 检查CAT会话状态        |
| `/api/diagnostic/smart-report`        | GET  | 生成综合智能诊断报告   |
| `/api/diagnostic/knowledge-tracing`   | GET  | 获取BKT知识掌握度      |
| `/api/diagnostic/misconceptions`      | GET  | 获取误区检测结果       |
| `/api/diagnostic/cognitive-profile`   | GET  | 获取认知画像           |

## 3. 智能诊断页面设计

### 3.1 诊断首页 (`/diagnostic`)

**页面结构**：

- 顶部：标题、描述、快速操作按钮
- 卡片区域：三种诊断方式选择
    - VARK学习风格测评（卡片1）
    - CAT自适应测试（卡片2）
    - 历史诊断记录（卡片3）

**设计要点**：

- 卡片悬停效果、渐变背景色区分不同诊断类型
- 显示预计时间、精准度标签
- 历史记录展示最近3次诊断结果摘要

### 3.2 CAT自适应测试页面 (`/diagnostic/cat`)

**页面结构**：

- 顶部进度条：显示当前进度、预计剩余题目数
- 左侧：题目内容、选项区域
- 右侧：实时能力估计仪表盘（θ值、标准误）
- 底部：答题计时、提交按钮

**核心交互**：

- 题目切换动画
- 实时能力变化曲线
- 答题耗时记录
- 进度预测（基于当前能力估计）

**设计要点**：

- 能力仪表盘采用半圆形进度条展示θ值（-3到+3范围）
- 标准误用颜色指示可靠性（绿色<0.35，黄色0.35-0.55，红色>0.55）
- 进度条显示已答题目数/预计总数

### 3.3 诊断报告页面 (`/diagnostic/report`)

**页面结构**：

- **顶部英雄区**：综合评分、评级（A/B/C/D/E）、核心结论
- **八维雷达图**：知识掌握、认知能力、学习韧性、逻辑推理、计算能力、概念理解、应用能力、元认知
- **知识热度图**：知识点掌握度矩阵（按学科分组）
- **Bloom认知金字塔**：六层认知层次可视化
- **误区卡片**：按严重度排序的误区分类
- **学习路径建议**：下一步行动列表

**设计要点**：

- 雷达图支持交互式悬停显示详细数值
- 热度图使用颜色渐变（绿色=掌握，红色=薄弱）
- 金字塔采用梯形堆叠展示各层掌握度
- 误区卡片支持点击展开查看样题和纠错建议

## 4. 学习画像页面设计

### 4.1 画像概览 (`/profile`)

**页面结构**：

- 顶部：画像标题、综合掌握度、核心标签
- 六维能力条形图：知识掌握、认知能力、学习态度、时间管理、问题解决、创新思维
- 优劣势对比卡片：优势能力、待提升能力
- 快速操作区：生成学习路径、问AI助手、重新校准

**设计要点**：

- 能力条形图采用渐变色填充
- 标签云展示核心特质
- 快捷按钮悬浮效果

### 4.2 认知分析 (`/profile/cognitive`)

**页面结构**：

- **认知属性追踪**：8项认知属性的掌握度进度条
    - 概念理解、公式应用、计算能力、逻辑推理、空间想象、归纳抽象、记忆检索、问题表征
- **Bloom层次分布**：六层柱状图
- **认知宽度评估**：当前到达的最高层次、各层次正确率

**设计要点**：

- 认知属性采用卡片网格布局
- 每个属性显示掌握度、置信度、样本数
- Bloom柱状图按层次高度递减排列

### 4.3 知识追踪 (`/profile/knowledge`)

**页面结构**：

- 学科筛选器：按学科过滤知识点
- 知识点列表：按掌握度排序
    - 显示：知识点名称、掌握度百分比、状态标签
- 掌握度趋势图：最近7天变化

**设计要点**：

- 知识点卡片颜色编码（绿=掌握，蓝=学习中，橙=薄弱，红=未掌握）
- 趋势图支持时间范围切换（7天/30天/90天）

### 4.4 误区管理 (`/profile/misconceptions`)

**页面结构**：

- 误区分类统计：饼图展示各类型占比
- 误区列表：按严重度排序
- 错题详情：点击误区查看具体错题
- 纠错建议：针对每个误区的专项训练建议

**设计要点**：

- 误区卡片显示类型、错误次数、严重度标签
- 错题展示用户答案与正确答案对比
- 纠错建议附带练习入口

## 7. API调用流程

### 7.1 CAT测试流程

```
用户进入 /diagnostic/cat
    ↓
POST /api/diagnostic/smart-start
    ↓
获取第一题，渲染题目
    ↓
用户选择答案并提交
    ↓
POST /api/diagnostic/smart-submit-answer
    ↓
更新能力估计、进度
    ↓
判断shouldContinue
    ├─ true → POST /api/diagnostic/smart-next-question → 继续答题
    └─ false → 跳转到诊断报告页面
```

### 7.2 报告生成流程

```
用户访问 /diagnostic/report
    ↓
GET /api/diagnostic/smart-report
    ↓
获取报告数据
    ↓
渲染报告组件
    ├─ 综合评分
    ├─ 雷达图
    ├─ 热度图
    ├─ 金字塔
    ├─ 误区卡片
    └─ 学习建议
```
