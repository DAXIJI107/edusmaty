## 1. 系统概述

### 1.1 现有架构

```
┌─────────────────────────────────────────────────────────┐
│                     EduSmart 平台                        │
├─────────────────────────────────────────────────────────┤
│  前端 SPA (edusmart-app.js ~5200行)                     │
│  ├─ render() → topbar() + view() + assistantFloat()     │
│  ├─ state.data._studentLocked 锁定标志                   │
│  └─ bindEvents() 委托事件绑定                            │
├─────────────────────────────────────────────────────────┤
│  后端 REST API (Express + MySQL)                        │
│  ├─ /api/teacher/*  教师端点 (requireTeacher)           │
│  └─ /api/student-paths/* 学生端点 (authenticateJWT)     │
├─────────────────────────────────────────────────────────┤
│  数据库 (MySQL edusmart_rebuild)                         │
│  ├─ teacher_learning_paths        (路径主表)             │
│  ├─ teacher_path_steps            (步骤表)               │
│  ├─ teacher_path_assignments      (分配表)               │
│  └─ teacher_path_step_progress    (进度表)               │
└─────────────────────────────────────────────────────────┘
```

### 1.2 当前能力 vs 目标能力

| 模块 | 当前已有 | 需要新增 |
|------|---------|---------|
| **教师-路径CRUD** | ✅ 创建/查看/更新/删除 | ⚠️ 需增强：拖拽排序、版本控制 |
| **教师-分配学生** | ✅ 勾选列表分配 | ❌ 搜索筛选、批量导入、移除学生 |
| **教师-发布流程** | ⚠️ 分配即发布 | ❌ 审核流程、定时发布、部分发布 |

| **学生-步骤完成** | ✅ text/video/quiz/code | ⚠️ 需增加 exercise 类型支持 |
| **学生-路径仪表盘** | ❌ 无 | ❌ 全新模块 |
| **学生-日历视图** | ❌ 无 | ❌ 全新模块 |
| **通知系统** | ❌ 无 | ❌ 全新模块 |

## 3. 数据库扩展设计

### 3.1 现有表（无需修改）

- `teacher_learning_paths` — 已完备
- `teacher_path_steps` — 已完备
- `teacher_path_assignments` — 已完备
- `teacher_path_step_progress` — 已完备

### 3.2 新增表：teacher_path_drafts（路径草稿/版本控制）

```sql
CREATE TABLE teacher_path_drafts (
    id INT AUTO_INCREMENT PRIMARY KEY,
    path_id INT NOT NULL,
    teacher_id INT NOT NULL,
    version INT DEFAULT 1,
    snapshot JSON NOT NULL COMMENT '完整路径快照 {name, description, subject, steps[]}',
    status ENUM('draft','review','approved','rejected') DEFAULT 'draft',
    review_comment TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    restored_at DATETIME,
    INDEX idx_path (path_id),
    INDEX idx_teacher (teacher_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.3 新增表：teacher_path_schedules（定时发布）

```sql
CREATE TABLE teacher_path_schedules (
    id INT AUTO_INCREMENT PRIMARY KEY,
    path_id INT NOT NULL,
    assignee_ids JSON NOT NULL COMMENT '目标学生ID列表',
    scheduled_at DATETIME NOT NULL,
    executed TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_path (path_id),
    INDEX idx_scheduled (scheduled_at, executed)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.4 新增表：notifications（通知系统）

```sql
CREATE TABLE notifications (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    type ENUM('path_assigned','path_completed','path_unlocked',
              'step_due','exam_scheduled','system') NOT NULL,
    title VARCHAR(200) NOT NULL,
    body TEXT,
    link VARCHAR(300),
    is_read TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user (user_id, is_read),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.5 现有表字段微调

```sql
-- teacher_path_assignments 新增字段
ALTER TABLE teacher_path_assignments 
    ADD COLUMN due_date DATETIME COMMENT '截止日期',
```

### 5.3 发布工作流

#### 5.3.1 发布状态机

```
  draft ──▶ review ──▶ approved ──▶ published
    │                    │
    └── rejected ◀───────┘
```

#### 5.3.2 发布流程

```
1. 教师创建路径 → status=draft
2. 教师提交审核 → status=review (可选, 未来扩展)
3. 审核通过 → status=approved, is_published=1
4. 选择学生并分配 → 学生端收到通知 + 锁定

定时发布:
1. 教师设置定时 → 存入 teacher_path_schedules
2. 服务端定时任务 (node-cron) 每分钟检查
3. 到期自动执行分配 → 更新 is_published + 创建通知
```

#### 5.3.3 部分发布

```javascript
// POST /api/teacher/paths/:id/assign
// 支持部分步骤发布
{
  student_ids: [1, 2, 3],
  start_step: 1,      // 从第几步开始
  end_step: 2,        // 到第几步结束（可选，默认全部）
  due_date: "2026-06-01"
}
```

| 事件 | 通知内容 | 接收方 |
|------|---------|--------|
