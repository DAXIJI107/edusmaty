## 2. 当前源码事实

### 2.1 前端架构过度集中

`apps/web/public/js/edusmart-app.js` 是一个超大型单文件 SPA，包含：

1. 路由识别：`routeToView()`，约第 285 行。
2. 数据加载：`loadData()` 到 `loadPathNotes()`，约第 352-1176 行。
3. 导航配置：`navGroups()`，约第 1501 行。
4. 页面渲染函数：`homeView()` 到 `agentCenterView()`，约第 1615-8050 行。
5. 事件绑定：`bindEvents()`，约第 8140 行后。
6. 最终视图映射：`views` 对象，约第 12419 行。

问题：页面、状态、API、事件和 UI 模板全部混在一个文件里，后续继续加功能会越来越难控制重复和样式漂移。

### 2.2 页面和功能确实重复

当前 `navGroups()` 中有 8 个一级导航组：

1. 首页
2. 学习中心
3. 诊断分析
4. 编程实践
5. 学习资源
6. 知识引擎
7. AI智能
8. 教师工作台

其中重复最明显的是：

1. 学习计划、专项练习、在线考试、阶段测试、复习巩固都在解决“今天学什么/如何验证/如何巩固”。
2. 路径页、Agent 学习中心、学习计划页、学生路径页都涉及学习路径或计划。
3. AI 助手、RAG 智能检索、Agent 研究中心、AI 闭环都在承载 AI 能力。
4. Obsidian 知识库、RAG 智能检索、计算机知识库、知识图谱、学习资产中心都在承载知识资产。
5. 智能诊断、学习画像、认知画像、知识掌握、误区检测、诊断报告属于同一画像中心，不适合散在多个入口。

### 2.3 有页面写了代码但没有正常显示

源码里存在“写了但前端不可达/不可正常渲染”的情况：

1. `knowledgeBaseView()` 存在，约第 7989 行，但 `views` 映射约第 12419 行没有包含 `knowledgeBase`。
2. `routeToView()` 会把 `/knowledge-base` 和 `/rag-knowledge` 转成 `knowledgeBase`，但最终 `views[state.view]` 找不到，实际回落到首页。
3. `studentPathView()` 存在，约第 1882 行。
4. `studentPathDashboardView()` 存在，约第 2004 行。
5. `views` 映射中虽然有 `studentPath: studentPathView`，但渲染前又执行 `if (state.view === "studentPath") state.view = "home";`，导致学生路径页面被强制隐藏。
6. `agentCenterView()` 当前只是返回 `knowledgeBaseView()`，但它和 `knowledgeBase` 的路由/导航关系不清晰。

结论：这不是单纯 UI 问题，而是信息架构和视图映射已经出现错位。

### 2.4 路由判断有重复和顺序风险

`routeToView()` 中存在重复判断：

1. `obsidian/vault`
2. `knowledge-base/rag-knowledge`
3. `rag-search/ragsearch`
4. `agent-center/agentcenter`

这些分支出现了两遍。并且 `p.includes("rag")` 会落到 `aiAssistant`，容易与 `ragSearch`、`knowledgeBase` 的意图冲突。

建议改为显式路径映射表，而不是继续用大量 `includes()`。

### 2.5 视觉系统有模板化倾向

`edusmart-pro.css` 当前主色包含：

1. `--blue`
2. `--blue-2`
3. `--violet`
4. `--purple`
5. `--pink`

再叠加大量 `card`、`metric-card`、`hero-row`、`grid-2`、`grid-3`、`quick-tile`，容易形成常见的“AI后台卡片墙”观感。

当前样式里也有一些局部重构痕迹，例如 `path-dashboard`、`pd-card`、`student-path-view` 等，但这些还没有统一成完整设计系统。

## 3. 新页面信息架构

建议把学生端一级导航重组为 6 个：

1. 学习驾驶舱：合并首页、今日计划、AI 今日闭环、复习到期、本周趋势。
2. 路径工作台：合并个性化路径、Agent 路径、学生路径、路径版本、手动维护。
3. AI 学习舱：合并 AI 助手、RAG 问答、错题讲解、笔记生成、费曼复盘。
4. 画像中心：合并智能诊断、学习画像、认知画像、知识掌握、误区检测、诊断报告。
5. 测评与练习：合并专项练习、阶段测试、在线考试、错题闭环。
6. 学习资产：合并智能笔记、知识库、知识图谱、Obsidian、资源、教程、视频。

教师端一级导航建议保留 4 个：

1. 教师控制台
2. 学生风险中心
3. 路径干预中心
4. 班级知识热力图

编程实践、团队项目、代码实验室可以作为“项目式学习”二级入口，不应抢占学习平台主导航。

## 6. 核心页面重做方案

### 6.1 学习驾驶舱

替换当前 `homeView()` 和 `studyPlanView()` 的分散体验。

首屏布局：

1. 左侧：画像罗盘，显示学习风格、专注时长、今日风险、画像可信度。
2. 中间：今日路径轨道，显示当前 active path 中被拆出的今日任务。
3. 右侧：AI 副驾驶，显示当前任务上下文、薄弱点、RAG 证据、下一步动作。
4. 底部：到期复习、错题补救、学习趋势。

验收：

1. 首页不再显示默认假任务。
2. 没有画像时提示先诊断。
3. 没有 active path 时提示生成路径。
4. AI 入口不是空白聊天，而是带当前任务上下文。

### 6.2 路径工作台

替换 `pathView()`，恢复并吸收 `studentPathView()`。

页面结构：

1. 路径轨道：Agent 节点、教师节点、用户手动节点统一显示。
2. 节点详情：资源、练习、预计时长、掌握目标。
3. 推荐依据：画像字段、错题证据、知识依赖、时间约束。
4. 编辑能力：新增、编辑、删除、排序。
5. 版本能力：保存快照、对比、回退。

必须修复：

1. 删除 `if (state.view === "studentPath") state.view = "home";`。
2. 将 `studentPathDashboardView()` 并入 `pathView()` 的 Tab。
3. 路由 `/student-paths/active`、`/student-paths/dashboard` 的数据进入同一页面。

### 6.3 AI 学习舱

替换单纯 `aiAssistantView()` 的聊天页定位。

合并：

1. `/api/app/tutor/message`
2. `/api/rag/ask`
3. `/api/app/notes/generate-card`
4. `/api/app/feynman/review`
5. `/api/ai-explain`

页面结构：

1. 上方上下文条：当前路径节点、今日任务、薄弱点、最近错题。
2. 中间对话：可追问、可引用 RAG 证据。
3. 右侧动作：生成练习、加入笔记、安排复习、更新画像候选。

### 6.4 画像中心

将 `diagnosticView()`、`profileView()`、`profileCognitiveView()`、`profileKnowledgeView()`、`profileMisconceptionsView()`、`diagnosticReportView()` 合并为一个页面内 Tab。

页面结构：

1. 画像摘要。
2. 诊断入口。
3. 知识掌握。
4. 误区检测。
5. 画像证据时间线。
6. 诊断报告。

### 6.5 测评与练习

将 `practice`、`test`、`onlineExam`、`exam` 合并。

页面结构：

1. 今日任务对应题目。
2. 薄弱点专项练习。
3. 阶段测试。
4. 错题本。
5. 错因分析。

### 6.6 学习资产

将 `assetView()` 作为主页面，吸收：

1. `smartNotesView()`
2. `knowledgeBaseView()`
3. `knowledgeGraphView()`
4. `obsidianView()`
5. `ragSearchView()` 的检索部分
6. `resources/tutorials/videos/problems` 的资源部分

必须修复：

1. `views` 映射加入 `knowledgeBase: knowledgeBaseView`。
2. 当 `state.view === "knowledgeBase"` 时调用 `loadKnowledgeBase()`。
3. RAG 问答部分从资产页跳转到 AI 学习舱，避免两个聊天入口。

## 7. 视觉设计方向

配套参考图：

`docs/assets/edusmart-source-redesign-style-reference.png`

设计关键词：

1. Apple 式：克制、清晰、留白、强层级、细腻材质。
2. Tesla 式：第一屏有明确视觉中心和推进感。
3. 教育平台独有：路径轨道、画像罗盘、知识地图、复习环、证据链。
4. 避免 AI 味：少用泛紫蓝渐变，少用同质卡片，少写功能介绍文案。

建议 token：

1. 背景：`#f6f8fb`、`#eef2f7`、`#0f141b`
2. 主文字：`#111827`
3. 次文字：`#667085`
4. 主强调：`#00a6d6`
5. 掌握成功：`#18b87a`
6. 风险提醒：`#f59e0b`
7. 边框：`rgba(17,24,39,.10)`
8. 深色面板：`#111820`

组件建议：

1. `LearningRail`：学习路径轨道。
2. `ProfileCompass`：画像罗盘。
3. `KnowledgeConstellation`：知识节点图。
4. `AICopilotPanel`：AI 副驾驶。
5. `EvidenceDrawer`：推荐依据抽屉。
6. `ReviewOrbit`：复习到期环。
